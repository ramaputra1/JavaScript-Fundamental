String & Number Convertion

Note:

Why we need this convertion? Sometimes the user input gonna be string when we need its supposed to be a number, or vice versa.
That is why when the input's data type does not meet our purpose, we need to convert it.

Ways:
1. parseInt(string) = whole number || when you have input (11kk) it still will be 11 but when you got (k11) it will be (NaN / Not a Number), so when you got number in front it's fine.
2. parseFloat(string) = decimal || same like parseInt
3. Number(string) = whole number or decimal || does not matter when you got the number front or back, the value gonna be always NaN

4. number.toString() = number to string



What is NaN?
It's a unique data, when you got not a string or not a number, for example: (11k), they got both number and character, It's NaN, or even "kkk" they are in " " and contains no number but char, so it's definetely NaN
When you try to do any operation with NaN, the output gonna be always NaN


That is why we got a thing to check if it's NaN or not, to prevent the wrong output (to be NaN always).
the method is: isNaN(number)
